### 3. Rewrite `apps/angular-spotify/src/main.ts`

v7 code (current):
```ts
import * as Sentry from '@sentry/angular';

Sentry.init({
  dsn: '...',
  integrations: [
    new Sentry.BrowserTracing({
      tracingOrigins: ['https://spotify.trungk18.com/'],
      routingInstrumentation: Sentry.routingInstrumentation
    })
  ],
  tracesSampleRate: 1.0
});
```

v8 replacement:
```ts
import * as Sentry from '@sentry/angular';

Sentry.init({
  dsn: '...',
  integrations: [
    Sentry.browserTracingIntegration()
  ],
  tracePropagationTargets: ['https://spotify.trungk18.com/'],
  tracesSampleRate: 1.0
});
```

Key changes: `new Sentry.BrowserTracing()` â†’ `Sentry.browserTracingIntegration()`; `tracingOrigins` â†’ top-level `tracePropagationTargets`; `Sentry.routingInstrumentation` is no longer passed explicitly (v8 wires it through the Angular router tracing automatically when you add `Sentry.TraceService` to the module providers).

Consult the Sentry v8 Angular migration guide if any detail here conflicts with the current SDK docs (this plan was written against v8 as of 2026-04-18 â€” use Context7 or WebFetch to verify current API).
