# Prep 2.4 — Remove Cypress

**Status:** Non-blocker (no peer-dep conflict; Cypress is dead weight)
**Parent plan:** `docs/plans/2026-04-18-angular-20-migration-plan.md`
**Branch:** `prep/cypress-remove`
**Runs in parallel with:** 2.1, 2.2, 2.3

---

## Goal

Remove Cypress and its related packages from the workspace. There are no e2e tests to preserve, so upgrading would be wasted work.

## Current state

**Current `package.json` entries (verified 2026-04-18):**
```json
"@nx/cypress": "18.2.0",
"cypress": "7.6.0",
"eslint-plugin-cypress": "2.15.1",
```

**Recon findings (verified 2026-04-18):**
- No `apps/*-e2e` folder exists
- No `cypress.config.{ts,js}` or `cypress.json` anywhere outside `node_modules/`
- No `*.cy.ts` or `*.cy.js` spec files exist in `apps/` or `libs/`
- References to `cypress` outside `node_modules/` exist only in `package.json`, `yarn.lock`, `nx.json`, and the migration plan itself

Cypress is unused. This is a removal, not an upgrade.

**Peer deps (verified):**
- `cypress@7.6.0`: none

## Scope

**In:**
1. Remove `cypress`, `@nx/cypress`, `eslint-plugin-cypress` from `package.json`
2. Update `nx.json` generator defaults so new apps/libs don't try to use Cypress
3. Remove any eslint config references to `eslint-plugin-cypress`
4. Confirm `yarn install` is clean and the build/test/lint targets still run

**Out:**
- Introducing Playwright or any other e2e runner
- Writing new tests
- Any Angular version bump

## Steps

### 1. Branch and sanity-check

```bash
git checkout -b prep/cypress-remove
yarn install
yarn build    # baseline green

# Re-verify nothing uses Cypress (expect empty output)
find . -path ./node_modules -prune -o -name "*.cy.ts" -print
find . -path ./node_modules -prune -o -name "*.cy.js" -print
find . -path ./node_modules -prune -o -name "cypress.config.*" -print
find . -path ./node_modules -prune -o -name "cypress.json" -print
ls apps/ | grep -i e2e
grep -rn "@nx/cypress" apps libs tools 2>/dev/null || true
grep -rn "eslint-plugin-cypress" apps libs tools 2>/dev/null || true
```

If any of these print non-empty output, **stop and ask the parent plan author** — the "Cypress is unused" premise no longer holds and the removal decision needs revisiting.

### 2. Remove dependencies

```bash
yarn remove cypress @nx/cypress eslint-plugin-cypress
```

This updates `package.json` and `yarn.lock`.

### 3. Clean up `nx.json`

Current `nx.json` has:
```json
"generators": {
  "@nx/angular:application": {
    "style": "scss",
    "linter": "eslint",
    "unitTestRunner": "jest",
    "e2eTestRunner": "cypress"
  },
  ...
}
```

Change `"e2eTestRunner": "cypress"` → `"e2eTestRunner": "none"` (or remove the key entirely).

Also check if any other generator in `nx.json` references cypress and remove those entries.

### 4. Clean up eslint config

Search for any reference to `eslint-plugin-cypress` or `plugin:cypress/recommended`:

```bash
grep -rn "cypress" .eslintrc.json apps libs --include=".eslintrc*" --include="eslint.config.*" 2>/dev/null
```

Remove any matching entries. Typical locations would be the root `.eslintrc.json` `plugins` / `extends` arrays, or per-project `.eslintrc.json` files generated for e2e projects (unlikely, given no e2e apps).

### 5. Final verification

```bash
grep -rn "cypress" package.json nx.json apps libs tools --include="*.json" --include="*.ts" --include="*.js" 2>/dev/null | grep -v "^Binary"
```

Expected: empty output (or only matches inside `yarn.lock` from transitive deps that happen to mention cypress in their description).

**PR body template:**
```markdown
## Why
- Cypress is listed as a dev dep but nothing consumes it — no `apps/*-e2e`, no `cypress.config.*`, no `*.cy.ts` specs anywhere outside `node_modules/`.
- Upgrading an unused dep through multiple breaking API changes (`cy.intercept`, `cy.session`, config format) is wasted work.
- Removing shrinks the dep tree and drops ~1GB of binary cache from CI.

## What changed
- Removed: `cypress@7.6.0`, `@nx/cypress@18.2.0`, `eslint-plugin-cypress@2.15.1`
- `nx.json`: `e2eTestRunner: "cypress"` → `e2eTestRunner: "none"`
- [any eslint config cleanups]

## Validation
- Build, test, lint all green
- No residual references to cypress in source or workspace config

## Re-introducing e2e later
If this project ever needs e2e coverage again, a fresh Cypress 13+ or Playwright install on the then-current Angular will be cleaner than upgrading through stale generations.

Part of the Angular 17 → 20 migration plan: `docs/plans/2026-04-18-angular-20-migration-plan.md`
```
