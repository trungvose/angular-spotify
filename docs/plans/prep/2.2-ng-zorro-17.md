## Goal

Close the peer-dep gap between `ng-zorro-antd@15.1.0` and Angular 17 by upgrading ng-zorro (and its icon companion) to v17. This is the highest-risk prep PR — ng-zorro touches ~20 files in the UI layer.

## Scope

**In:**

2. Ensure `@ant-design/icons-angular@^17` is installed (explicit or via ng-zorro's deps)
3. Run the ng-zorro schematic for v16 and v17 migrations
4. Fix all breaking changes across v16 and v17

## Steps

### 1. Branch and baseline

```bash
git checkout -b prep/ng-zorro-17
yarn install
yarn build    # baseline green
yarn start    # quick manual smoke before we change anything
```

### 2. Upgrade

Try the Angular schematic first (it applies ng-zorro's own migration transforms):

```bash
yarn ng update ng-zorro-antd@17 --allow-dirty --force
```

If the schematic balks (it might, because our `@angular/core@17.3.2` is already past ng-zorro 17's paired Angular), fall back to manual:

```bash
yarn add ng-zorro-antd@^17 @ant-design/icons-angular@^17
```

### 3. Breaking-change sweep

Major categories to watch for across v16 → v17:

- **Theme tokens**: ng-zorro 16 consolidated the theme system. Any SCSS variable overrides in `styles.scss` referencing `@nz-*` variables may need renaming. Check ng-zorro's v16 and v17 changelog.
- **Form control API**: `nz-form-control` signatures tightened. Templates with `[nzErrorTip]` or custom validators may need updates.
- **Modal service**: `NzModalService.create()` signature; some options were renamed.
- **Select component**: `nz-select` with custom templates — watch for the `.ant-select-selection-item::after` strut bug (documented in the Jira Clone Part 7 migration).
- **Deprecated inputs**: v16 and v17 removed several deprecated inputs (`nzIconType` on some components, etc.)

Reference the official ng-zorro migration guides:
- v16: https://github.com/NG-ZORRO/ng-zorro-antd/blob/master/CHANGELOG.md (v16.0.0 section)
- v17: https://github.com/NG-ZORRO/ng-zorro-antd/blob/master/CHANGELOG.md (v17.0.0 section)

### 4. Manual verification per usage

For each file from the grep output, open the corresponding page/component in `yarn start`, interact with it, confirm no visual regressions or console errors.

**Priority smoke paths** (based on app structure):
- Login / unauthorized modal
- Player controls (volume, playback, visualization toggle)
- Lyrics toggle
- Social share popover
- Any `nz-message` / `nz-notification` toasts that fire on actions
