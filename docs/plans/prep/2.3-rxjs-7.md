# Prep 2.3 — RxJS 6 → 7

**Status:** Non-blocker (no peer-dep conflict; EOL hygiene)
**Parent plan:** `docs/plans/2026-04-18-angular-20-migration-plan.md`
**Branch:** `prep/rxjs-7`
**Runs in parallel with:** 2.1, 2.2, 2.4 — smallest PR of the four

---

## Goal

Move off EOL RxJS 6. The peer-dep data shows every Angular version 17 → 20 accepts `rxjs@^6.5.3 || ^7.4.0`, so this isn't a migration blocker — but RxJS 6 has been EOL since 2022, `.toPromise()` is deprecated, and landing modern RxJS before the Angular series keeps each subsequent PR focused.

## Current state

**Current `package.json`:**
```json
"rxjs": "6.6.6",
```

**Source recon (verified via grep on 2026-04-18):**
- `.toPromise()` call sites: **zero** in `apps/` and `libs/`
- No `Observable.create(` usage expected (verify)

This means the migration is essentially a version bump. Surprising, but welcome. The sweep is mostly defensive.

**Peer deps (verified):**
- `rxjs@6.6.6`: no peer deps declared
- `@angular/core@17.3.2`: `rxjs: ^6.5.3 || ^7.4.0`
- `@ngrx/store@17.0.1`: (check — historically `^6.5.3 || ^7.5.0`)

## Scope

**In:**
1. Bump `rxjs` to `^7.8.0`
2. Sweep for `.toPromise()` and replace with `firstValueFrom` / `lastValueFrom` (expected: zero hits, but double-check)
3. Sweep for `Observable.create(` → `new Observable(`
4. Sweep for array-form `combineLatest([...])` — valid in both, but verify no `combineLatest(a, b, c)` legacy form remains
5. Confirm tests still pass (NgRx + RxJS combos are the typical friction point)

**Out:**
- RxJS 8 (not released as stable; ignore)
- Refactors from `pipe().subscribe()` to signals (separate initiative)
- Any Angular version bump

## Steps

### 1. Branch and baseline

```bash
git checkout -b prep/rxjs-7
yarn install
yarn build    # baseline green
```

### 2. Bump

```bash
yarn add rxjs@^7.8.0
```

### 3. Sweep (expected no-op but do it anyway)

```bash
# Any leftover .toPromise()?
grep -rn "\.toPromise(" apps libs --include="*.ts"

# Observable.create anywhere?
grep -rn "Observable\.create(" apps libs --include="*.ts"

# Legacy combineLatest spread form?
grep -rn "combineLatest(" apps libs --include="*.ts" | grep -v "combineLatest(\["
```

If any hits, fix:
- `x.toPromise()` → `firstValueFrom(x)` or `lastValueFrom(x)` depending on whether x is expected to emit once or continues to emit
- `Observable.create(fn)` → `new Observable(fn)`
- `combineLatest(a, b, c)` → `combineLatest([a, b, c])`

### 4. Full build + test

```bash
yarn nx run-many -t build
yarn nx run-many -t test
yarn nx run-many -t lint
```

RxJS 7 removed some deprecated operators (`partition` as a free function, etc.) — if any of these surface, replace with the pipeable equivalent.
